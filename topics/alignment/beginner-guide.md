# 对齐（Alignment）科普版

> 面向零基础读者的对齐技术入门指南

## 一句话概括

**对齐让模型从"能说话"变成"会说话"，学会有帮助、诚实、无害。**

## 从一个问题开始

预训练后的模型已经会"说人话"了，但还有问题：

- 可���输出有害内容
- 可能编造事实（幻觉）
- 可能不遵循指令

**对齐就是教模型"怎么做个好助手"。**

## 对齐的三个目标（3H）

| 目标 | 含义 | 示例 |
|------|------|------|
| **Helpful** 有帮助 | 真正解决用户问题 | 给出可执行的答案 |
| **Honest** 诚实 | 不编造、承认不确定 | "我不确定，建议查证" |
| **Harmless** 无害 | 不产生有害内容 | 拒绝危险请求 |

## Stage 1: SFT（监督微调）

### 概念

用人工编写的"问答对"训练模型：

```
用户：什么是光合作用？
助手：光合作用是植物利用阳光、水和二氧化碳制造养分的过程...
```

### 类比

就像给学生看"标准答案"，让他们学习怎么回答问题。

### 数据要求

- 高质量的人工编写
- 覆盖多种任务类型
- 格式统一、风格一致

## Stage 2: 奖励模型

### 概念

训练一个"评分员"模型，给回答打分：

```
问题：推荐一个北京景点

回答 A：故宫是必去的...（详细、有帮助）  → 高分
回答 B：不知道                        → 低分
回答 C：北京很无聊                    → 中等分数
```

### 训练方法

让人类对多个回答进行排序：

```
问题：写一首关于春天的诗

回答排序（人类标注）：
1. 回答 C（最好） > 回答 A > 回答 B（最差）

奖励模型学习：C > A > B
```

## Stage 3: RLHF / DPO

### RLHF（基于人类反馈的强化学习）

```
┌─────────────────────────────────────────────────────────────┐
│                     RLHF 流程                                │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   ┌──────────┐     ┌──────────┐     ┌──────────┐          │
│   │  Policy  │ ──▶ │  Reward  │ ──▶ │  Update  │          │
│   │  Model   │     │  Model   │     │  Policy  │          │
│   └──────────┘     └──────────┘     └──────────┘          │
│        │                │                │                 │
│        │                │                │                 │

│   生成回答          给出分数        调整参数               │

│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### DPO（直接偏好优化）

**更简单的方法！** 不需要单独训练奖励模型：

```
┌─────────────────────────────────────────────────────────────┐
│                      DPO 流程                                │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   直接用偏好数据优化模型：                                    │
│                                                             │
│   问题：推荐一本编程书                                        │
│                                                             │
│   好的回答 (yw): "推荐《Python编程》..."                      │
│   差的回答 (yl): "编程很难"                                   │
│                                                             │
│   目标：增大 P(yw) - P(yl)                                   │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### RLHF vs DPO

| 对比 | RLHF | DPO |
|------|------|-----|
| 复杂度 | 高（需要 3 个模型） | 低（只需 1 个模型） |
| 稳定性 | 较难调参 | 更稳定 |
| 效果 | 成熟 | 相当或更好 |
| 计算成本 | 高 | 低 |

    ▼

    │
    ▼

    ▼

### Constitutional AI

用"宪法"（规则）自动生成反馈：

```
宪法规则：
1. 不要输出有害内容
2. 要诚实
3. 要有帮助

模型自己检查输出是否符合规则
→ 减少对人类标注的依赖
```

## 总结

| 概念 | 通俗理解 |
|------|----------|
| SFT | 看标准答案学习 |
| 奖励模型 | 训练一个评分员 |
| RLHF | 用奖励训练模型（像训练狗狗） |
| DPO | 直接用偏好数据优化 |
| 3H 原则 | 有帮助、诚实、无害 |
