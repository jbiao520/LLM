# 注意力变体（Attention Variants）科普版

> 面向零基础读者的注意力优化技术入门指南

## 一句话概括

**注意力变体通过共享和优化，让模型更快、更省内存，同时保持效果。**

## 从一个问题开始

Transformer 的注意力机制很强大，但有一个问题：**太吃资源了！**

- 计算量：O(n²) —— 长度翻倍，计算量翻四倍
- 内存：需要存储 n×n 的注意力矩阵

**怎么解决这个问题？** 答案就是注意力变体。

## MHA vs MQA vs GQA

### 类比：会议讨论

| 方式 | 类比 | 特点 |
|------|------|------|
| **MHA** (Multi-Head Attention) | 每个人都有独立的问题和笔记 | 最准确，但资源消耗大 |
| **MQA** (Multi-Query Attention) | 所有人共享问题和笔记 | 最省资源，但可能损失精度 |
| **GQA** (Grouped Query Attention) | 分组共享，组内共享问题和笔记 | 平衡：省资源且保持精度 |

## Flash Attention

### 问题：内存带宽瓶颈

标准注意力需要多次读写 HBM（高带宽内存）：

```
GPU 内存层次:
┌─────────────────────────────────────────────┐
│ HBM (高带宽内存) - 大但慢                    │
│   ↓ 读/写                                   │
│ SRAM (片上缓存) - 小但快                     │
│   ↓ 计算                                    │
│ GPU 核心                                    │
└─────────────────────────────────────────────┘

标准注意力: HBM → SRAM → 计算 → HBM (多次)
Flash Attention: HBM → SRAM → 计算 → HBM (一次)
```

### 解决方案：分块计算

```
┌─────────────────────────────────────────────────────────────────┐
│                    Flash Attention 分块计算                     │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  标准注意力:                                                    │
│  计算整个 S = QK^T → 存储到 HBM → 读取 → softmax → 存储         │
│  (内存: O(n²))                                                 │
│                                                                 │
│  Flash Attention:                                               │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │ for block_Q in Q:                                       │   │
│  │     for block_K, block_V in K, V:                       │   │
│  │         在 SRAM 中计算局部注意力                         │   │
│  │         增量更新输出（不需要存储完整注意力矩阵）          │   │
│  │     写入最终输出到 HBM                                   │   │
│  └─────────────────────────────────────────────────────────┘   │
│  (内存: O(n))                                                  │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 效果

| 指标 | 标准注意力 | Flash Attention |
|------|-----------|-----------------|
| 内存 | O(n²) | O(n) |
| 速度 | 基准 | 2-4x 更快 |
| 精度 | 基准 | 完全相同 |

## Sliding Window Attention

### 概念

只关注附近的 token，而不是所有 token：

```
完整注意力 (窗口大小 = 全部):
┌─────────────────────────────────────────────┐
│ ● ● ● ● ● ● ● ● ● ● ● ● ● ● ● ● ● ● ● ●  │
│ 每个位置可以看到所有其他位置                  │
└─────────────────────────────────────────────┘

滑动窗口注意力 (窗口大小 = 3):
┌─────────────────────────────────────────────┐
│           ● ● ●                             │
│         ● ● ●                               │
│       ● ● ●                                 │
│     ● ● ●                                   │
│   ● ● ●                                     │
│ ● ● ●                                       │
│ 只能看到附近 3 个位置                        │
└─────────────────────────────────────────────┘
```

### 应用

- Mistral：使用滑动窗口注意力
- 长文本生成：减少计算量

## 总结

| 技术 | 解决的问题 | 方法 | 效果 |
|------|-----------|------|------|
| **GQA** | KV Cache 太大 | 分组共享 KV | 内存减少 4x |
| **MQA** | KV Cache 太大 | 完全共享 KV | 内存减少 8x |
| **Flash Attention** | 内存带宽瓶颈 | 分块计算 | 速度 2-4x |
| **Sliding Window** | 计算量 O(n²) | 只看附近 | 计算量 O(n) |
