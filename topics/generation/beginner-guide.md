# 文本生成（Generation）科普版

> 面向零基础读者的文本生成入门指南

## 一句话概括

**文本生成就像"词语接龙"，模型每次预测下一个最可能的词，一步步组成完整的回答。**

## 从一个问题开始

当你问 ChatGPT "今天天气怎么样？"，它是怎么生成回答的？

难道模型脑子里已经有一个完整的答案？**不是的！**

模型是**一个字一个字**生成的：
1. 看到 "今天天气怎么样？" → 预测 "今天"
2. 看到 "今天天气怎么样？今天" → 预测 "天气"
3. 看到 "今天天气怎么样？今天天气" → 预测 "不错"
4. ...继续下去

**就像你玩"词语接龙"游戏一样！**

## 核心概念：下一个词预测

### 概率分布

模型每次会计算**所有可能的下一个词的概率**：

```
输入："今天天气"
模型输出概率：
  "不错"   → 30%
  "很好"   → 25%
  "糟糕"   → 15%
  "一般"   → 10%
  "晴朗"   → 8%
  ...（其他所有词分剩余 12%）
```

### 选择方式

有了概率，怎么选词？

| 方法 | 做法 | 特点 |
|------|------|------|
| 贪婪 | 选概率最高的 | 最保守，但可能无聊 |
| 随机 | 按概率随机选 | 有创意，但可能跑题 |
| 温度 | 调整概率分布 | 平衡保守和创意 |

## 温度（Temperature）：创意的"旋钮"

### 类比：餐厅选择

假设你在一个城市选餐厅：

- **温度 = 0**：永远去评分最高的那家（确定性）
- **温度 = 0.5**：大概率去高分餐厅，偶尔尝试新店
- **温度 = 1.0**：按真实评分分布选择
- **温度 = 2.0**：完全随机，什么都可能选

### 数学直觉

温度会"压平"或"尖锐化"概率分布：

```
原始概率：    温度 = 0.5:    温度 = 2.0:
A: 50%        A: 70%         A: 35%
B: 30%   →    B: 25%    →    B: 30%
C: 20%        C: 5%          C: 35%
```

- **低温**：概率差异放大，更确定
- **高温**：概率差异缩小，更随机

### 实���效果

| 温度 | 输出特点 | 适用场景 |
|------|---------|---------|
| 0 - 0.3 | 保守、稳定、可预测 | 代码生成、事实问答 |
| 0.5 - 0.7 | 平衡 | 一般对话 |
| 0.8 - 1.0 | 有创意、多样 | 创意写作、头脑风暴 |
| 1.0+ | 很随机、可能不连贯 | 实验、探索 |

## Top-K：只从最好的 K 个中选

### 概念

只保留概率最高的 K 个词，其他直接排除：

```
原始概率：           Top-K = 3：
A: 40%              A: 40%
B: 30%              B: 30%
C: 20%              C: 20%
D: 8%      →        (排除)
E: 2%               (排除)
```

然后在这 K 个中按概率选择。

### 为什么有用？

- 避免选到完全不该出现的词
- 同时保持一定的多样性
- 常见设置：K = 40 或 K = 50

## Top-P（Nucleus Sampling）：按概率累计选择

### 概念

选择最少的词，使得它们的概率之和 ≥ P：

```
原始概率：           Top-P = 0.9：
A: 50%              A: 50%  ↘
B: 30%              B: 30%   → 90%，够了
C: 10%              (排除)
D: 8%               (排除)
E: 2%               (排除)
```

### 与 Top-K 的区别

| 方法 | 思路 | 优点 | 缺点 |
|------|------|------|------|
| Top-K | 固定数量 | 简单 | 可能太少或太多 |
| Top-P | 固定概率 | 自适应 | 计算稍复杂 |

**实际使用中，Top-P 通常效果更好。**

## 组合使用

现代 LLM API 通常允许同时设置：

```python
# OpenAI API 示例
response = client.chat.completions.create(
    model="gpt-4",
    messages=[{"role": "user", "content": "讲个笑话"}],
    temperature=0.7,   # 创意程度
    top_p=0.9,         # 核采样
    # top_k 通常不直接暴露
)
```

### 推荐组合

| 场景 | Temperature | Top-P |
|------|-------------|-------|
| 代码生成 | 0.2 | 0.9 |
| 问答/摘要 | 0.3 | 0.9 |
| 日常对话 | 0.7 | 0.9 |
| 创意写作 | 0.9 | 0.95 |
| 头脑风暴 | 1.0 | 0.99 |

## Beam Search：多路径探索

### 概念

不是只生成一条路径，而是同时探索多条"最有希望的路径"：

```
输入: "翻译成英文："

Beam = 1 (贪婪):     Beam = 3:
                     ┌→ "The" → "cat" → ...
"Translate" → ...    ├→ "The" → "dog" → ...
                     └→ "A" → "cat" → ...
```

最后选择概率最高的那条完整路径。

### 适用场景

- 机器翻译（需要高质量）
- 代码补全
- 任何需要"最优"答案的场景

### 缺点

- 计算成本高（要生成多条）
- 可能产生重复
- 不适合创意任务

## 重复惩罚（Repetition Penalty）

### 问题

模型有时会"卡住"，重复同样的内容：

```
"今天天气不错，天气不错，天气不错，天气不错..."
```

### 解决方案

对已经出现过的词施加"惩罚"：

```
原始概率：           惩罚后 (已出现 "天气")：
天气: 40%           天气: 10% (÷4)
不错: 30%     →     不错: 40% (提升)
晴朗: 20%           晴朗: 27%
```

常见设置：惩罚系数 = 1.1 到 1.5

## 停止条件

模型怎么知道什么时候停止？

1. **遇到结束符** (`<\|end\|>`, `</s>`)
2. **达到最大长度** (max_tokens)
3. **遇到停止词** (如 "###", "---")

## 总结

| 概念 | 通俗理解 |
|------|----------|
| 下一个词预测 | 词语接龙 |
| Temperature | 创意旋钮，低=保守，高=随机 |
| Top-K | 只从最好的 K 个里选 |
| Top-P | 选到累计概率达到 P 为止 |
| Beam Search | 同时探索多条路径 |
| 重复惩罚 | 防止说车轱辘话 |
