# Word Embedding 科普版

> 面向零基础读者的词嵌入入门指南

## 一句话概括

**词嵌入就是把每个词变成一串数字，让意思相近的词，数字也相近。**

## 从一个问题开始

想象你是一个外国人，刚学中文。你看到两个句子：

- "我喜欢吃**苹果**"
- "我喜欢吃**香蕉**"

你怎么知道"苹果"和"香蕉"是类似的东西？

作为人类，你理解它们都是水果。但计算机只看到两个不同的字符串，它们没有任何关系。

**词嵌入就是要解决这个问题：让计算机也能"理解"词语之间的相似性。**

## 核心思想：用数字表示词

### 最简单的方法：编号

给每个词一个编号：
- 苹果 = 1
- 香蕉 = 2
- 汽车 = 3
- 飞机 = 4

**问题**：苹果(1)和香蕉(2)的编号相近，但香蕉(2)和汽车(3)也相近！编号的相近程度不等于语义的相近程度。

### 更好的方法：用多个数字（向量）

与其用一个数字，不如用多个数字来描述一个词的"特征"。

想象我们用 3 个特征来描述每个词：

| 词语 | 是水果吗？ | 是交通工具吗？ | 能吃吗？ |
|------|-----------|---------------|---------|
| 苹果 | 1.0 | 0.0 | 1.0 |
| 香蕉 | 1.0 | 0.0 | 1.0 |
| 汽车 | 0.0 | 1.0 | 0.0 |
| 飞机 | 0.0 | 1.0 | 0.0 |

现在，苹果和香蕉的数字几乎一样 `[1, 0, 1]`，而它们和汽车 `[0, 1, 0]` 完全不同！

这就是**词嵌入**的核心：用一组数字（向量）来表示每个词，让语义相似的词有相似的数字表示。

## 实际的词嵌入

现实中，我们不用 3 个数字，而是用 **几百个数字**（比如 300 维）。

这些数字不是人工定义的"是水果吗"这种特征，而是**让机器自己从大量文本中学出来的**。

机器会观察：
- "苹果"经常和"吃"、"水果"、"甜"一起出现
- "香蕉"也经常和"吃"、"水果"、"甜"一起出现
- 所以它们的向量应该很相似

## 一个神奇的特性：语义运算

词嵌入有一个惊人的特性——可以做"词语运算"：

```
国王 - 男人 + 女人 ≈ 王后
```

向量之间的加减法竟然能反映语义关系！这说明词嵌入真的"学会"了词语的含义。

## 经典方法

### Word2Vec (2013)

Google 提出的方法，让词嵌入真正流行起来。核心思想很简单：
- 看一个词**周围**有哪些词
- 语境相似的词，意思也相似

### GloVe (2014)

Stanford 提出的方法，结合了全局统计信息。

## 总结

| 概念 | 通俗理解 |
|------|----------|
| 词嵌入 | 把词变成数字串 |
| 向量 | 那串数字 |
| 相似度 | 两串数字的"距离" |
| 维度 | 数字串的长度 |
