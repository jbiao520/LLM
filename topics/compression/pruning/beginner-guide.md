# 剪枝（Pruning）科普版

> 面向零基础读者的模型剪枝入门指南

## 一句话概括

**剪枝就像修剪树木，剪掉"没用的枝叶"（参数），让模型更精简高效。**

## 从一个问题开始

训练好的神经网络里有很多参数，但不是所有参数都重要：

- 很多参数接近 0，几乎不影响输出
- 有些神经元很少被激活
- 删掉它们，模型可能照样工作！

**问题：哪些参数可以剪？剪多少？怎么剪？**

│   │   /|\       │                │   /|\       │               │
│   │  / | \      │                │  / | \      │               │
│   │ /  |  \     │   剪掉枯枝     │ /  |  \     │               │

│   │    |    \   │                │    |     \  │               │

## 核心概念

### 重要 vs 不重要

怎么判断一个参数重要不重要？

**最简单方法：看绝对值大小**

```
权重矩阵：
┌────────────────────────────────────────┐
│  2.5   0.01  -1.8   0.001              │
│ -0.02  3.1    0.005  0.8               │
│  1.2  -0.001  2.9   -0.03              │
│  0.008  0.9  -0.009  1.5               │
└────────────────────────────────────────┘

重要（大）：2.5, 3.1, 2.9, 1.5...
不重要（小）：0.01, 0.001, 0.005, 0.008...
```

### 稀疏度（Sparsity）

剪枝后，很多参数变成 0：

$$\text{稀疏度} = \frac{\text{零值参数数}}{\text{总参数数}}$$

- 50% 稀疏度 = 一半参数是 0
- 90% 稀疏度 = 90% 参数是 0

## 两种剪枝方式

### 1. 非结构化剪枝（Unstructured）

**随机删除单个参数**

```
原始矩阵：              剪枝后（50%稀疏）：
┌─────────────┐        ┌─────────────┐
│ a b c d e   │        │ a 0 c 0 e   │
│ f g h i j   │   ──▶  │ 0 g 0 i 0   │
│ k l m n o   │        │ k 0 m 0 0   │
└─────────────┘        └─────────────┘

优点：剪得最精确，精度损失小
缺点：需要特殊硬件加速（普通 GPU 不快）
```

### 2. 结构化剪枝（Structured）

**删除整行/整列/整个滤波器**

```
原始网络：              剪枝后（删除神经元2）：
┌─────────────┐        ┌─────────────┐
│ 神经元 1    │        │ 神经元 1    │
│ 神经元 2    │   ──▶  │ 神经元 3    │
│ 神经元 3    │        │ 神经元 4    │
│ 神经元 4    │        └─────────────┘
└─────────────┘

优点：普通硬件就能加速
缺点：精度损失可能更大
```

### 对比

| 方式 | 精度 | 加速 | 难度 |
|------|------|------|------|
| 非结构化 | 高 | 需要特殊硬件 | 低 |
| 结构化 | 较低 | 任何硬件 | 高 |

## 剪枝流程

```
┌─────────────────────────────────────────────────────────────────┐
│                     剪枝三步曲                                    │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   Step 1: 训练模型                                               │
│   ┌──────────────────────────────────────────────────────────┐  │
│   │  正常训练，得到一个"过参数化"的模型                        │  │
│   │  模型学到了很多"冗余"参数                                  │  │
│   └──────────────────────────────────────────────────────────┘  │
│                              │                                  │
│                              ▼                                  │
│   Step 2: 评估重要性                                            │
│   ┌──────────────────────────────────────────────────────────┐  │
│   │  对每个参数/神经元评估"重要性"                             │  │
│   │  方法：绝对值大小、梯度大小、对输出的影响...               │  │
│   └──────────────────────────────────────────────────────────┘  │
│                              │                                  │
│                              ▼                                  │
│   Step 3: 剪枝 + 微调                                           │
│   ┌──────────────────────────────────────────────────────────┐  │
│   │  剪掉不重要的参数 → 精度下降 → 微调恢复精度                 │  │
│   │  可能需要多次迭代                                          │  │
│   └──────────────────────────────────────────────────────────┘  │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 剪枝粒度

从细到粗：

| 粒度 | 剪什么 | 加速效果 |
|------|--------|----------|
| 权重级 | 单个参数 | 需特殊硬件 |
| 内核级 | 一组参数 | 中等 |
| 通道级 | 整个通道 | 好 |
| 层级 | 整层 | 最好 |

```
权重级：         内核级：        通道级：        层级：
┌───────┐      ┌───────┐     ┌───────┐     ┌───────┐
│a 0 c 0│      │a b c d│     │a b c d│     │a b c d│
│0 g 0 i│      │e f g h│     │e f g h│     │       │
│k 0 m 0│      │       │     │       │     │ 层删除 │
└───────┘      └───────┘     └───────┘     └───────┘
 随机零          块零          整行零         整层零
```

## 彩票假说（Lottery Ticket Hypothesis）

一个有趣的发现：

> **一个过参数化的网络里，藏着一个"中奖"的小网络，从头训练它也能达到原网络的性能。**

**意义：**
1. 剪枝不只是"删除"，而是"发现"好结构
2. 剪枝后的小网络可以重新训练，而不只是微调

## 实际效果

| 模型 | 原始参数 | 剪枝后 | 稀疏度 | 精度保持 |
|------|----------|--------|--------|----------|
| ResNet-50 | 25M | 5M | 80% | ~99% |
| BERT-Base | 110M | 33M | 70% | ~98% |
| GPT-2 | 1.5B | 0.3B | 80% | ~95% |

## 总结

| 概念 | 通俗理解 |
|------|----------|
| 剪枝 | 删掉不重要的参数 |
| 稀疏度 | 零值参数的比例 |
| 非结构化 | 随机删单个参数 |
| 结构化 | 删整行/整列 |
| 彩票假说 | 大网络里藏着小网络 |
| 微调 | 剪枝后恢复精度 |
