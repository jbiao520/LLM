# 知识蒸馏（Knowledge Distillation）深入版

> 面向有机器学习基础读者的知识蒸馏深度指南

## 1. 蒸馏的理论基础

### 1.1 软标签的数学原理

传统训练使用硬标签（one-hot）：

$$y_{hard} = [0, 1, 0, 0]$$

蒸馏使用软标签，来自 Teacher 的 softmax：

$$p_i^T = \frac{\exp(z_i/T)}{\sum_j \exp(z_j/T)}$$

其中：
- $z_i$：第 $i$ 类的 logit
- $T$：温度参数
- $p_i^T$：温度为 $T$ 时的软概率

### 1.2 为什么软标签更好？

**信息论视角：**

硬标签的熵：
$$H(y_{hard}) = 0$$

软标签的熵：
$$H(p^T) = -\sum_i p_i^T \log p_i^T > 0$$

软标签包含更多信息（暗知识/Dark Knowledge）。

**例子：**
```
图像分类：这是一只狗
硬标签：[狗=1, 猫=0, 狼=0, 马=0]

Teacher 软标签：[狗=0.9, 狼=0.08, 猫=0.015, 马=0.005]
隐含信息：狗和狼长得像！
```

### 1.3 温度的作用

温度 $T$ 控制分布的"平滑度"：

$$\frac{\partial p_i}{\partial z_i} = \frac{p_i(1-p_i)}{T}$$

- **T = 1**：标准 softmax
- **T > 1**：分布更平滑，小概率类的信息更明显
- **T → ∞**：均匀分布

**最优温度：** 通常 $T \in [2, 5]$，需要调参。

## 2. 蒸馏损失函数

### 2.1 标准 KD 损失

$$L_{KD} = \alpha \cdot L_{soft} + (1-\alpha) \cdot L_{hard}$$

其中：

**软目标损失（蒸馏损失）：**
$$L_{soft} = T^2 \cdot KL(p^T_{teacher} \| p^T_{student})$$

**硬目标损失（任务损失）：**
$$L_{hard} = CE(y_{true}, p^S_{T=1})$$

**为什么乘 $T^2$？**

因为 softmax 在温度 $T$ 下的梯度缩放了 $1/T$ 倍，需要补偿：
$$\frac{\partial L_{soft}}{\partial z_i} \propto \frac{1}{T}$$

### 2.2 KL 散度展开

$$KL(P \| Q) = \sum_i P(i) \log \frac{P(i)}{Q(i)}$$

对于蒸馏：
$$KL(p^T_{teacher} \| p^T_{student}) = \sum_i p^T_{teacher,i} \log \frac{p^T_{teacher,i}}{p^T_{student,i}}$$

### 2.3 余弦嵌入损失（替代方案）

$$L_{cosine} = 1 - \frac{h_s \cdot h_t}{\|h_s\| \|h_t\|}$$

直接对齐隐藏层表示，不使用概率。

## 3. 蒸馏变体

### 3.1 中间层蒸馏

不仅蒸馏最终输出，还蒸馏中间层：

$$L_{feature} = \sum_l \| \phi_l(h_s^l) - h_t^l \|^2$$

其中 $\phi_l$ 是适配函数（如线性投影）。

**Hint Learning** (Romero et al., 2014)
- 学生模仿教师的中间层特征
- 需要设计适配层

### 3.2 注意力蒸馏

蒸馏注意力图：

$$L_{attn} = \sum_l \| A_s^l - A_t^l \|_F^2$$

**DistilBERT** 使用此方法保留语言理解能力。

### 3.3 渐进式蒸馏

逐步增加蒸馏权重：

$$\alpha(t) = \alpha_{start} + (\alpha_{end} - \alpha_{start}) \cdot \frac{t}{T_{max}}$$

早期更多依赖硬标签，后期更多依赖软标签。

## 4. 特殊蒸馏方法

### 4.1 自蒸馏（Self-Distillation）

模型蒸馏自己：

1. 训练一个大模型
2. 用大模型的软标签重新训练
3. 有时甚至能用更小的模型！

**原理：** 软标签提供正则化效果。

### 4.2 在线蒸馏（Online Distillation）

多个模型同时训练，互相学习：

```
┌─────────────────────────────────────────────────────────────────┐
│                     在线蒸馏                                     │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   Model A ◄──────────────▶ Model B                             │
│       ▲                         ▲                               │
│       │        互相学习          │                               │
│       ▼                         ▼                               │
│   Model C ◄──────────────▶ Model D                             │
│                                                                 │
│   集成输出作为"软教师"                                           │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 4.3 数据无关蒸馏

不依赖原始训练数据，只用教师模型的输出：

1. 生成合成数据（或使用公开数据）
2. 用教师模型打标签
3. 训练学生模型

**应用：** 模型压缩时无法访问原始数据。

## 5. LLM 蒸馏实践

### 5.1 Alpaca 方法

1. 用 GPT-4 生成 52K 指令-回答对
2. 用这些数据微调 LLaMA 7B
3. 学生模型获得接近 GPT-4 的指令跟随能力

### 5.2 Vicuna 方法

1. 收集 ShareGPT 对话数据
2. 用这些数据训练 LLaMA
3. 训练了 370K 对话

### 5.3 蒸馏的局限

| 问题 | 原因 |
|------|------|
| 能力天花板 | 学生无法超越教师 |
| 领域偏移 | 训练数据与推理数据分布不同 |
| 蒸馏鸿沟 | 某些能力难以转移（如推理） |

## 6. 实践建议

### 6.1 温度选择

| 任务类型 | 推荐温度 |
|----------|----------|
| 分类任务 | 2-5 |
| 生成任务 | 1-2 |
| 推理任务 | 1 |

### 6.2 损失权重

| 阶段 | $\alpha$ (蒸馏权重) |
|------|---------------------|
| 初期 | 0.5 |
| 中期 | 0.7 |
| 后期 | 0.9 |

### 6.3 学生模型选择

| 压缩比 | 准确率保持 |
|--------|------------|
| 2x | ~99% |
| 4x | ~95% |
| 10x | ~85% |
