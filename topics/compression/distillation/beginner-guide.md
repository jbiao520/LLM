# 知识蒸馏（Knowledge Distillation）科普版

> 面向零基础读者的知识蒸馏入门指南

## 一句话概括

**知识蒸馏让小模型学会大模型的"智慧"，用更少的参数达到接近大模型的效果。**

## 从一个问题开始

你训练了一个超强的 GPT-4 级别模型，但：

- 模型太大（1T 参数），部署成本高
- 推理太慢，用户体验差
- 想在手机上运行？不可能！

**怎么办？** 答案就是知识蒸馏。

## 核心概念

### Teacher（教师模型）

- 已经训练好的大模型
- 知识丰富，但"重"
- 作为"知识来源"

### Student（学生模型）

- 待训练的小模型
- 容量小，但"轻"
- 目标是接近 Teacher 的能力

### 软标签（Soft Labels）

这是蒸馏的关键！

**硬标签 vs 软标签：**

```
问题：这张图是什么动物？

硬标签（传统训练）：
┌─────────────────────────────────────────┐
│ 猫: 0%     狗: 100%    鸟: 0%    兔: 0% │
└─────────────────────────────────────────┘
只知道"这是狗"

软标签（蒸馏训练）：
┌─────────────────────────────────────────┐
│ 猫: 5%     狗: 85%     鸟: 2%    兔: 8% │
└─────────────────────────────────────────┘
知道"这主要是狗，但也有点像猫和兔"
```

**软标签包含更多信息！** Teacher 不仅告诉答案，还告诉"为什么"。

## 温度（Temperature）的作用

温度控制"软"的程度：

```
原始输出（logits）: [10, 2, 1, 0.5]

温度 T = 1（正常 softmax）:
[0.999, 0.001, 0.000, 0.000]  ← 几乎是硬标签

温度 T = 4（高温）:
[0.85, 0.08, 0.05, 0.02]      ← 更软，更多信息

温度 T = 10（超高温）:
[0.60, 0.20, 0.12, 0.08]      ← 太软，可能丢失信息
```

**类比：**
- 低温度 = 老师只给答案
- 高温度 = 老师详细解释每个选项的优缺点

## 蒸馏损失函数

蒸馏的损失 = 两部分：

1. **蒸馏损失**：学生匹配老师的软标签
2. **任务损失**：学生预测正确答案

$$L_{total} = \alpha \cdot L_{distill} + (1-\alpha) \cdot L_{task}$$

其中 $\alpha$ 通常是 0.5-0.9。

## 蒸馏的效果

| 模型 | 参数量 | 准确率 |
|------|--------|--------|
| Teacher (BERT-Large) | 340M | 92% |
| Student (DistilBERT) | 66M | 87% |
| Student 从头训练 | 66M | 75% |

**蒸馏让学生模型提升 12%！**

## 实际应用

### DistilBERT

- 保留了 97% 的 BERT 能力
- 参数减少 40%
- 速度提升 60%

### DistilGPT-2

- 保留了 95% 的 GPT-2 能力
- 参数减少 50%
- 更适合边缘设备部署

### Alpaca/Vicuna

- 用 GPT-4 的输出训练小模型
- 让 7B 模型获得接近 GPT-4 的对话能力

## 总结

| 概念 | 通俗理解 |
|------|----------|
| Teacher | 老师，知识渊博的大模型 |
| Student | 学生，正在学习的小模型 |
| 软标签 | 老师的"思考过程"，不只是答案 |
| 温度 | 控制"软"的程度，高温=更多信息 |
| 蒸馏损失 | 学生要模仿老师的输出分布 |
