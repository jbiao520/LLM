## 一句话概括

**多智能体协作让多个 AI"角色"分工合作，像团队一样解决复杂任务。**

## 从一个问题开始

一个复杂任务：���发一个网站

单 Agent 难以处理：
- 需求分析需要沟通技巧
- 前端开发需要 UI 知识
- 后端开发需要架构设计
- 测试需要找 bug 的能力

**解决方案：** 让多个专业 Agent 组成团队！

## 核心概念

### 1. 角色（Role）

每个 Agent 有特定的角色和职责。

| 角色 | 职责 | System Prompt 示例 |
|------|------|-------------------|
| 规划者 | 分解任务、制定计划 | "你是一个项目规划专家..." |
| 研究员 | 搜索信息、整理资料 | "你是一个信息研究专家..." |
| 开发者 | 编写代码、实现功能 | "你是一个资深程序员..." |
| 审查者 | 检查质量、发现问题 | "你是一个代码审查专家..." |

### 2. 通信（Communication）

Agent 之间如何交流：

```

│   ┌─────────────────────────────────────────────────────────┐   │
│   │  Agent A ──── message ────▶ Agent B                     │   │
│   │  Agent B ──── message ────▶ Agent A                     │   │
│   └─────────────────────────────────────────────────────────┘   │
```

### 3. 协作模式

| 模式 | 描述 | 适用场景 |
|------|------|----------|
| **顺序** | 一个接一个执行 | 流程化任务 |
| **并行** | 同时执行多个任务 | 独立子任务 |
| **层级** | 主管分配给下属 | 复杂任务分解 |
| **对等** | 平等讨论、投票 | 需要共识的任务 |

## 实际框架

### AutoGen (Microsoft)

```python
from autogen import AssistantAgent, UserProxyAgent

# 创建助手
assistant = AssistantAgent(
    name="assistant",
    system_message="你是一个有帮助的AI助手。"
)

# 创建用户代理
user_proxy = UserProxyAgent(
    name="user_proxy",
    human_input_mode="NEVER",
    code_execution_config={"work_dir": "coding"}
)

# 开始对话
user_proxy.initiate_chat(
    assistant,
    message="写一个冒泡排序"
)
```

### CrewAI

```python
from crewai import Agent, Task, Crew

# 定义 Agent
researcher = Agent(
    role="研究员",
    goal="研究最新技术",
    backstory="你是一个技术研究员",
    allow_delegation=False
)

writer = Agent(
    role="作家",
    goal="写技术文章",
    backstory="你是一个技术作家",
    allow_delegation=True
)

# 定义任务
research_task = Task(
    description="研究 LLM 最新进展",
    agent=researcher
)

write_task = Task(
    description="写一篇 LLM 综述",
    agent=writer
)

# 组建团队
crew = Crew(
    agents=[researcher, writer],
    tasks=[research_task, write_task]
)

# 执行
crew.kickoff()
```

## 协作的好处

| 好处 | 说明 |
|------|------|
| **专业化** | 每个 Agent 专注自己擅长的 |
| **并行化** | 多个任务同时进行 |
| **鲁棒性** | 一个 Agent 失败，其他可以补救 |
| **可扩展** | 需要新能力就加新 Agent |
| **可解释** | 每个步骤都有明确的 Agent 负责 |

## 协作的挑战

| 挑战 | 说明 |
|------|------|
| **通信开销** | Agent 之间频繁交互 |
| **协调复杂** | 需要管理 Agent 之间的关系 |
| **一致性** | 多个 Agent 的输出可能冲突 |
| **调试困难** | 难以追踪问题来源 |
| **成本** | 多个 LLM 调用成本更高 |

## 总结

| 概念 | 通俗理解 |
|------|----------|
| Agent | AI 员工，有自己的职责 |
| Role | Agent 的角色定位 |
| Communication | 员工之间的交流方式 |
| 顺序协作 | 流水线作业 |
| 层级协作 | 主管-下属模式 |
| 辩论协作 | 头脑风暴、投票决策 |
