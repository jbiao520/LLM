# 记忆系统（Memory）科普版

> 面向零基础读者的记忆系统入门指南

## 一句话概括

**记忆系统让 AI 像人一样"记住"对话历史和知识，不仅限于当前对话。**

## 从一个问题开始

你和一个助手聊天：

```
你: 我叫小明
助手: 你好小明！

... 100 句对话后 ...

你: 我叫什么？
助手: ???（忘了）
```

**问题：** LLM 的上下文有限，聊长了就"忘记"之前的内容。

**解决：** 记忆系统！

## 类比：人的记忆

```
┌─────────────────────────────────────────────────────────────────┐
│                     人类记忆 vs AI 记忆                          │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   人类记忆:                                                     │
│   ┌──────────────────────────────────────────────────────────┐  │
│   │  短期记忆: 刚才说的话、当前在想的事                        │  │
│   │  长期记忆: 昨天的事、学过的知识、人生经历                  │  │
│   │  工作记忆: 正在处理的信息                                  │  │
│   └──────────────────────────────────────────────────────────┘  │
│                                                                 │
│   AI 记忆:                                                      │
│   ┌──────────────────────────────────────────────────────────┐  │
│   │  对话历史 (Context): 当前对话的消息列表                    │  │
│   │  向量数据库 (Vector DB): 存储和检索长期知识                │  │
│   │  用户画像 (Profile): 用户的偏好、习惯                      │  │
│   └──────────────────────────────────────────────────────────┘  │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 记忆类型

### 1. 短期记忆（对话历史）

就是最近的对话内容，放在 LLM 的 Context 里。

```
当前对话:
┌─────────────────────────────────────────┐
│ 用户: 今天天气不错                       │
│ 助手: 是的，很适合出门！                 │
│ 用户: 推荐个活动                         │
│ 助手: 可以去公园散步...                  │
│ 用户: [当前问题]                         │
└─────────────────────────────────────────┘
        ↑ 这部分就是短期记忆
```

**特点：**
- 精确，不丢失信息
- 容量有限（4K-128K tokens）
- 超过限制需要截断或压缩

### 2. 长期记忆（向量数据库）

把重要信息存到向量数据库，需要时检索。

```
┌─────────────────────────────────────────────────────────────────┐
│                     长期记忆流程                                  │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   存储:                                                         │
│   ┌─────────────────────────────────────────────────────────┐   │
│   │ "用户喜欢喝咖啡" → Embedding → [0.2, 0.5, ...]          │   │
│   │                                 ↓                       │   │
│   │                           Vector DB                     │   │
│   └─────────────────────────────────────────────────────────┘   │
│                                                                 │
│   检索:                                                         │
│   ┌─────────────────────────────────────────────────────────┐   │
│   │ 查询: "用户喜欢什么饮料？"                               │   │
│   │         ↓ Embedding                                     │   │
│   │ 相似度搜索 → 找到 "用户喜欢喝咖啡"                       │   │
│   │         ↓                                               │   │
│   │ 返回: "用户喜欢喝咖啡"                                   │   │
│   └─────────────────────────────────────────────────────────┘   │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 3. 工作记忆（摘要）

把长对话压缩成摘要，保留关键信息。

```
原始对话（1000字）:
┌─────────────────────────────────────────┐
│ 用户: 我想学编程...                      │
│ 助手: 好的，从Python开始...              │
│ 用户: 我已经学了变量...                  │
│ ... (更多对话)                           │
│ 用户: 下一步学什么？                     │
└─────────────────────────────────────────┘
            ↓ 压缩
摘要（100字）:
┌─────────────────────────────────────────┐
│ 用户正在学习Python编程，已经掌握了       │
│ 变量、循环、条件语句，现在询问下一步     │
│ 学习建议。                               │
└─────────────────────────────────────────┘
```

## 记忆操作

### 写入（存储）

```
┌─────────────────────────────────────────────────────────────────┐
│                     记忆写入                                      │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   用户消息: "我最近在学机器学习"                                 │
│                                                                 │
│   1. 判断是否值得记忆                                           │
│      - 包含个人偏好？ → 是 ✓                                   │
│      - 事实信息？ → 是 ✓                                       │
│      - 临时问题？ → 否                                         │
│                                                                 │
│   2. 提取记忆条目                                               │
│      {                                                         │
│        "content": "用户在学习机器学习",                        │
│        "type": "user_status",                                  │
│        "timestamp": "2024-01-15"                               │
│      }                                                         │
│                                                                 │
│   3. 存入向量数据库                                             │
│      embedding = embed("用户在学习机器学习")                   │
│      db.store(embedding, metadata)                            │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 读取（检索）

```
┌─────────────────────────────────────────────────────────────────┐
│                     记忆检索                                      │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   当前问题: "推荐学习资源"                                       │
│                                                                 │
│   1. 生成查询向量                                               │
│      query_embedding = embed("推荐学习资源")                   │
│                                                                 │
│   2. 相似度搜索                                                 │
│      results = db.search(query_embedding, top_k=5)            │
│                                                                 │
│   3. 检索结果:                                                  │
│      - "用户在学习机器学习" (sim: 0.85)                        │
│      - "用户会Python" (sim: 0.72)                              │
│      - "用户是初学者" (sim: 0.68)                              │
│                                                                 │
│   4. 注入到提示                                                 │
│      "用户背景: 在学习机器学习，会Python，是初学者..."          │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 常见向量数据库

| 数据库 | 特点 | 适用场景 |
|--------|------|----------|
| Chroma | 轻量、易用 | 小型应用 |
| Pinecone | 托管、高性能 | 生产环境 |
| Milvus | 开源、可扩展 | 大规模应用 |
| FAISS | Meta 出品 | 本地部署 |
| Weaviate | 功能丰富 | 企业级应用 |

## 记忆管理策略

### 滑动窗口

只保留最近 N 条消息：

```
[消息1] [消息2] [消息3] [消息4] [消息5]
                              ↑ 保留
[消息2] [消息3] [消息4] [消息5] [消息6]  ← 新消息进来后
```

### 优先级保留

重要消息优先保留：

```
重要: 用户偏好、事实信息、关键决策
次要: 一般对话
丢弃: 问候、闲聊
```

### 混合策略

```
┌─────────────────────────────────────────┐
│ 最近 N 条消息（滑动窗口）                │
│ +                                       │
│ 重要信息摘要（长期记忆）                 │
│ +                                       │
│ 用户画像（持久存储）                     │
└─────────────────────────────────────────┘
```

## 总结

| 概念 | 通俗理解 |
|------|----------|
| 短期记忆 | 最近说的话（在 Context 里） |
| 长期记忆 | 存在数据库里的知识 |
| 向量数据库 | 按"相似度"找信息的仓库 |
| Embedding | 把文字变成数字向量 |
| 摘要 | 把长对话压缩成短文 |
