# 规划与推理（Planning）科普版

> 面向零基础读者的规划与推理入门指南

## 一句话概括

**规划让 LLM 像"聪明人"一样思考，把复杂问题拆成小步骤，一步步解决。**

## 从一个问题开始

问题："帮我策划一次日本旅行"

**普通 LLM：** 直接给出一个可能混乱的回答

**有规划能力的 Agent：**
1. 先拆解：日期、预算、景点、住宿、交通...
2. 分步解决：确定日期 → 选择城市 → 预订酒店...
3. 检查一致性：预算够不够？时间来得及吗？

## 核心推理模式

### 1. Chain-of-Thought (CoT)

**思维链：一步步思考**

```
问题: 小明有5个苹果，给了小红2个，又买了3个，现在有几个？

普通回答: 6个 ❌（可能算错）

CoT 回答:
小明原来有 5 个
给了小红 2 个，剩下 5-2=3 个
又买了 3 个，现在 3+3=6 个
答案: 6个 ✓
```

### 2. Tree-of-Thought (ToT)

**思维树：探索多条路径**

```
┌─────────────────────────────────────────────────────────────────┐
│                     思维树示例                                    │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   问题: 选择旅行目的地                                          │
│                                                                 │
│                    ┌───────────┐                               │
│                    │   开始    │                               │
│                    └─────┬─────┘                               │
│                          │                                      │
│            ┌─────────────┼─────────────┐                        │
│            ▼             ▼             ▼                        │
│        ┌───────┐    ┌───────┐    ┌───────┐                    │
│        │ 东京  │    │ 大阪  │    │ 京都  │                    │
│        │评分:8 │    │评分:7 │    │评分:9 │                    │
│        └───┬───┘    └───┬───┘    └───┬───┘                    │
│            │            │            │                          │
│            ▼            ▼            ▼                          │
│      [继续评估]    [继续评估]    [继续评估]                     │
│            │            │            │                          │
│            └────────────┴────────────┘                          │
│                          │                                      │
│                          ▼                                      │
│                    选择京都 ✓                                   │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 3. Self-Consistency

**自洽性：多路径投票**

```
同一个问题，用不同思路解答多次:

思路 1: 答案是 A
思路 2: 答案是 A
思路 3: 答案是 B
思路 4: 答案是 A
思路 5: 答案是 A

投票结果: A (4票) > B (1票)
最终答案: A
```

## 任务分解方法

### 分解策略

| 策略 | 描述 | 适用场景 |
|------|------|----------|
| 按步骤 | 顺序分解 | 流程类任务 |
| 按模块 | 功能分解 | 复杂系统 |
| 按难度 | 先易后难 | 层次任务 |
| 按依赖 | 先无依赖 | 并行任务 |

### 分解示例

```
原始任务: "写一篇关于气候变化的文章"

分解:
├── 1. 研究阶段
│   ├── 1.1 搜索气候变化数据
│   ├── 1.2 查找科学报告
│   └── 1.3 收集案例
├── 2. 大纲阶段
│   ├── 2.1 确定主题
│   ├── 2.2 列出章节
│   └── 2.3 分配篇幅
├── 3. 写作阶段
│   ├── 3.1 写引言
│   ├── 3.2 写正文
│   └── 3.3 写结论
└── 4. 审核阶段
    ├── 4.1 检查逻辑
    ├── 4.2 检查语法
    └── 4.3 检查引用
```

## 规划与执行的平衡

| 类型 | 特点 | 适用场景 |
|------|------|----------|
| 先规划后执行 | 完整规划，然后执行 | 确定性任务 |
| 边规划边执行 | 动态调整计划 | 不确定性高 |
| 完全反应式 | 无规划，直接执行 | 简单任务 |

## 总结

| 概念 | 通俗理解 |
|------|----------|
| Chain-of-Thought | 一步步想，写在纸上 |
| Tree-of-Thought | 多想几种可能，选最好的 |
| Self-Consistency | 多算几次，少数服从多数 |
| 任务分解 | 大事化小，逐个击破 |
| 反思 | 做完检查，发现不足再改进 |
