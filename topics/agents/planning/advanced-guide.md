# 规划与推理（Planning）深入版

> 面向有 ML 基础读者的规划与推理深度指南

## 1. Chain-of-Thought (CoT)

### 1.1 数学形式化

给定问题 $Q$，CoT 生成一系列推理步骤 $R_1, R_2, ..., R_n$ 和最终答案 $A$：

$$P(A, R | Q) = \prod_{i=1}^{n} P(R_i | Q, R_{<i}) \cdot P(A | Q, R)$$

### 1.2 Zero-shot CoT

```
Prompt: "Let's think step by step."
```

无需示例，直接触发推理能力。

### 1.3 Few-shot CoT

```
Q: 小明有5个苹果...
A: 小明原来有5个，给了2个剩下3个，又买了3个...
   答案: 6个

Q: [新问题]
A:
```

### 1.4 自动 CoT (Auto-CoT)

自动生成推理链示例：
1. 对问题聚类
2. 每类选代表问题
3. 用 LLM 生成推理链
4. 作为 few-shot 示例

## 2. Tree-of-Thought (ToT)

### 2.1 搜索算法

```
┌─────────────────────────────────────────────────────────────────┐
│                     ToT 搜索策略                                 │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   BFS (广度优先):                                               │
│   ┌─────────────────────────────────────────────────────────┐   │
│   │ Level 0:              [根]                              │   │
│   │ Level 1:         [A]   [B]   [C]                        │   │
│   │ Level 2:     [D][E] [F][G] [H][I]                       │   │
│   │ 逐层扩展，保留所有候选                                    │   │
│   └─────────────────────────────────────────────────────────┘   │
│                                                                 │
│   DFS (深度优先):                                               │
│   ┌─────────────────────────────────────────────────────────┐   │
│   │ [根] → [A] → [D] (评估)                                 │   │
│   │              ↓ (如果不好)                                │   │
│   │        回溯到 [A] → [E]                                 │   │
│   │ 深入探索，及时剪枝                                        │   │
│   └─────────────────────────────────────────────────────────┘   │
│                                                                 │
│   Best-First (最佳优先):                                        │
│   ┌─────────────────────────────────────────────────────────┐   │
│   │ 优先队列: 按评分排序                                      │   │
│   │ 每次扩展评分最高的节点                                    │   │
│   └─────────────────────────────────────────────────────────┘   │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 2.2 评估函数

$$V(s) = \text{LLM\_Evaluate}(\text{state } s)$$

使用 LLM 自身评估每个状态：
- "这个推理步骤是否合理？"
- "这条路径有希望得到正确答案吗？"

### 2.3 代码实现

```python
def tot_search(problem, max_depth=5, beam_width=3):
    beam = [Node(state=initial_state(problem))]

    for depth in range(max_depth):
        candidates = []
        for node in beam:
            # 生成多个后继
            thoughts = generate_thoughts(node.state, k=beam_width)
            for thought in thoughts:
                new_state = apply_thought(node.state, thought)
                score = evaluate_state(new_state)
                candidates.append(Node(state=new_state, score=score))

        # 保留 top-k
        beam = sorted(candidates, key=lambda x: x.score)[:beam_width]

        # 检查是否找到解
        if any(is_solution(n.state) for n in beam):
            return extract_solution(beam)

    return best_solution(beam)
```

## 3. Self-Consistency

### 3.1 采样策略

从同一问题采样多条推理路径：

$$\{A_1, A_2, ..., A_k\} \sim P(A | Q, \text{temperature} > 0)$$

### 3.2 聚合方法

**投票：**
$$A^* = \arg\max_a \sum_{i=1}^{k} \mathbb{1}[A_i = a]$$

**加权投票：**
$$A^* = \arg\max_a \sum_{i=1}^{k} w_i \cdot \mathbb{1}[A_i = a]$$

权重可以基于：
- 推理链长度
- 置信度分数
- 验证结果

## 4. 任务分解算法

### 4.1 HTN (Hierarchical Task Network)

```
高层任务: 写文章
    ├── 子任务: 研究主题
    │       ├── 搜索资料
    │       └── 整理笔记
    ├── 子任务: 写大纲
    ├── 子任务: 写正文
    │       ├── 写引言
    │       ├── 写主体
    │       └── 写结论
    └── 子任务: 审核修改
```

### 4.2 分解条件

任务 $T$ 可以分解当：
$$T \rightarrow \{T_1, T_2, ..., T_n\}$$

满足：
- 原子性：每个 $T_i$ 可直接执行或继续分解
- 完备性：$\bigcup T_i$ 覆盖 $T$ 的所有需求
- 顺序性：$T_i$ 之间的依赖关系明确

## 5. 反思机制

### 5.1 Reflexion 框架

```python
def reflexion_loop(task, max_iterations=3):
    trajectory = []

    for i in range(max_iterations):
        # 1. 行动
        action = agent.act(trajectory)
        result = execute(action)
        trajectory.append((action, result))

        # 2. 反思
        reflection = llm.generate(
            f"分析刚才的行动和结果：{trajectory}\n"
            f"哪里做得好？哪里需要改进？"
        )

        # 3. 检查是否完成
        if is_complete(result):
            return result

        # 4. 更新记忆
        memory.add(reflection)

    return best_result(trajectory)
```

### 5.2 自我评估

$$\text{score} = \text{LLM}(Q, A, \text{"评估这个回答的质量 (1-10)"})$$

用于：
- 选择最佳答案
- 决定是否继续迭代
- 识别需要改进的部分
