# 工具调用（Tool Use）科普版

> 面向零基础读者的工具调用入门指南

## 一句话概括

**工具调用让 LLM 从"只会说话"变成"能干实事"，通过调用外部工具完成计算、搜索、操作等任务。**

## 从一个问题开始

你问 ChatGPT："今天北京天气怎么样？"

**问题：** GPT 的训练数据截止到某个时间点，它不知道"今天"的天气！

**解决：** 让 GPT 能够"上网查"——这就是工具调用。

## 核心概念

### 1. 工具（Tool）

工具是 LLM 可以调用的外部函数或 API。

| 工具类型 | 示例 | 功能 |
|----------|------|------|
| 搜索 | Google Search, Bing | 获取实时信息 |
| 计算 | Calculator, Python | 精确计算 |
| 数据库 | SQL Query | 查询数据 |
| 代码执行 | Code Interpreter | 运行代码 |
| API 调用 | Weather API, Stock API | 获取特定数据 |

### 2. 工具定义

告诉 LLM 有哪些工具可用：

```json
{
  "name": "get_weather",
  "description": "获取指定城市的天气信息",
  "parameters": {
    "city": {
      "type": "string",
      "description": "城市名称，如'北京'"
    }
  }
}
```

## ReAct 模式

**Re**asoning + **Act**ing = 思考 + 行动

```
┌─────────────────────────────────────────────────────────────────┐
│                     ReAct 循环                                   │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   问题: "马斯克的公司市值是多少？"                               │
│                                                                 │
│   Thought 1: 我需要先确定马斯克有哪些公司                        │
│   Action 1: search("Elon Musk companies")                       │
│   Observation 1: Tesla, SpaceX, X (Twitter), Neuralink...      │
│                                                                 │
│   Thought 2: Tesla 是上市公司，我需要查它的市值                  │
│   Action 2: search("Tesla market cap")                          │
│   Observation 2: Tesla市值约 5000亿美元                         │
│                                                                 │
│   Thought 3: SpaceX 不是上市公司，没有公开市值                   │
│   Thought 4: 我可以回答 Tesla 的市值了                          │
│                                                                 │
│   Answer: 马斯克的主要上市公司 Tesla 市值约 5000 亿美元...       │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## Function Calling

OpenAI 等平台提供的标准化工具调用接口。

### 请求格式

```json
{
  "model": "gpt-4",
  "messages": [{"role": "user", "content": "北京天气"}],
  "tools": [
    {
      "type": "function",
      "function": {
        "name": "get_weather",
        "parameters": {
          "type": "object",
          "properties": {
            "city": {"type": "string"}
          }
        }
      }
    }
  ]
}
```

### 响应格式

```json
{
  "choices": [{
    "message": {
      "tool_calls": [{
        "function": {
          "name": "get_weather",
          "arguments": "{\"city\": \"北京\"}"
        }
      }]
    }
  }]
}
```

## 常见工具类型

| 工具 | 描述 | 示例应用 |
|------|------|----------|
| **搜索引擎** | 搜索互联网信息 | 实时问答、事实核查 |
| **计算器** | 执行数学计算 | 复杂计算、财务分析 |
| **代码执行** | 运行代码 | 数据分析、绘图 |
| **数据库查询** | 查询结构化数据 | 业务报表、用户查询 |
| **文件操作** | 读写文件 | 文档处理、数据分析 |
| **API 调用** | 调用外部服务 | 订票、支付、发邮件 |

## 工具调用的挑战

| 挑战 | 说明 |
|------|------|
| **选择错误** | LLM 可能选错工具 |
| **参数错误** | 参数格式或值不正确 |
| **调用顺序** | 需要多个工具时顺序可能混乱 |
| **结果理解** | LLM 可能误解工具返回结果 |
| **安全风险** | 恶意工具调用可能造成危害 |

## 实际应用示例

### 代码解释器

```
用户: "帮我分析这个CSV文件并画图"
       │
       ▼
LLM: [读取文件] → [Python分析] → [生成图表] → 返回结果
```

### 多工具协作

```
用户: "查一下苹果公司最新股价，并计算如果买入100股需要多少钱"
       │
       ▼
LLM: [调用股价API] → [获取股价 $180] → [计算器: 180×100] → "需要 $18,000"
```

## 总结

| 概念 | 通俗理解 |
|------|----------|
| 工具 | LLM 的"手"，能执行实际操作 |
| Function Calling | 标准化的工具调用接口 |
| ReAct | 边想边做：思考→行动→观察→重复 |
| 工具定义 | 告诉 LLM 有什么工具可用 |
| 工具调用 | LLM 决定用哪个工具、传什么参数 |
