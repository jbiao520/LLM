# 分布式训练（Distributed Training）科普版

> 面向零基础读者的分布式训练入门指南

## 一句话概括

**分布式训练让多台机器像团队一样协作，共同完成单机无法完成的大模型训练任务。**

## 从一个问题开始

想象你要训练一个 GPT-4 级别的模型：

- 模型参数：1 万亿（1T）
- 单张 GPU 显存：80GB
- 问题：模型根本放不下！

**怎么解决？** 答案就是分布式训练。

## 类比：团队协作

| 场景 | 单机 | 分布式 |
|------|------|--------|
| 搬家 | 一个人搬完所有东西 | 多人分工，各搬一部分 |
| 盖房子 | 一个建筑师做所有事 | 团队协作：设计、施工、装修 |
| 训练大模型 | 单张 GPU | 多张 GPU/多台机器 |

**核心思想：分而治之！**

## 三种并行策略

### 1. 数据并行（Data Parallelism）

**类比：多个人同时做相同的任务，各自处理不同的数据。**

```
┌───────────────────��─────────────────────────────────────────────┐
│                      数据并行                                    │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   总数据 Batch = 64                                             │
│         │                                                       │
│         ├─── GPU 0: 模型副本 + Batch 16                         │
│         ├─── GPU 1: 模型副本 + Batch 16                         │
│         ├─── GPU 2: 模型副本 + Batch 16                         │
│         └─── GPU 3: 模型副本 + Batch 16                         │
│                     │                                           │
│                     ▼                                           │
│              汇总梯度，同步更新                                   │
│                                                                 │
│   特点：每张卡有完整模型，处理不同数据                              │
│   限制：模型必须能放入单张 GPU                                    │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 2. 张量并行（Tensor Parallelism）

**类比：把一个大任务拆成多个小任务，分给不同专家同时做。**

```
┌─────────────────────────────────────────────────────────────────┐
│                      张量并行                                    │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   矩阵乘法 Y = X × W                                             │
│                                                                 │
│   ┌─────────────────────────────────────────────┐               │
│   │              原始权重矩阵 W (4096 × 4096)    │               │
│   └─────────────────────────────────────────────┘               │
│                      │                                          │
│            ┌─────────┴─────────┐                                │
│            ▼                   ▼                                │
│   ┌──────────────┐    ┌──────────────┐                         │
│   │ GPU 0: W[:,  │    │ GPU 1: W[:,  │                         │
│   │   0:2048]    │    │   2048:4096] │                         │
│   └──────────────┘    └──────────────┘                         │
│         │                    │                                  │
│         ▼                    ▼                                  │
│   Y_0 = X × W_0       Y_1 = X × W_1                            │
│         │                    │                                  │
│         └────────┬───────────┘                                  │
│                  ▼                                              │
│           Y = [Y_0, Y_1] 拼接                                   │
│                                                                 │
│   特点：切分层内计算，需要高速通信                                 │
│   要求：GPU 之间需要 NVLink 等高带宽连接                          │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 3. 流水线并行（Pipeline Parallelism）

**类比：工厂流水线，每个工人负责一段工序。**

```
┌─────────────────────────────────────────────────────────────────┐
│                      流水线并行                                  │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   32 层 Transformer 分成 4 段：                                  │
│                                                                 │
│   ┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐     │
│   │  GPU 0  │───▶│  GPU 1  │───▶│  GPU 2  │───▶│  GPU 3  │     │
│   │ 层 1-8  │    │ 层 9-16 │    │ 层 17-24│    │ 层 25-32│     │
│   └─────────┘    └─────────┘    └─────────┘    └─────────┘     │
│                                                                 │
│   数据像流水一样依次流过各 GPU                                     │
│                                                                 │
│   问题：GPU 空闲时间（气泡）                                      │
│   ┌──────────────────────────────────────────────────────────┐  │
│   │ GPU 0: ████████░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░ │  │
│   │ GPU 1: ░░░░░░░░████████░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░ │  │
│   │ GPU 2: ░░░░░░░░░░░░░░░░████████░░░░░░░░░░░░░░░░░░░░░░░ │  │
│   │ GPU 3: ░░░░░░░░░░░░░░░░░░░░░░░░████████░░░░░░░░░░░░░░░ │  │
│   └──────────────────────────────────────────────────────────┘  │
│   █ = 工作，░ = 等待（气泡）                                      │

```

## 三种并行对比

| 并行方式 | 切分对象 | 通信频率 | 适用场景 |
|----------|----------|----------|----------|
| 数据并行 | 数据 | 每步梯度同步 | 模型能放入单卡 |
| 张量并行 | 层内权重 | 每层计算 | 超大模型，高带宽 |
| 流水线并行 | 层间分配 | 层间传递 | 深层模型 |

## ZeRO：零冗余优化

**问题：** 数据并行中，每张卡都存完整模型，太浪费！

**ZeRO 的解决方案：** 分片存储

```
┌─────────────────────────────────────────────────────────────────┐
│                        ZeRO 优化                                 │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   原始数据并行：每张卡存全部                                       │
│   ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐          │
│   │ 模型参数  │ │ 模型参数  │ │ 模型参数  │ │ 模型参数  │          │
│   │ 梯度     │ │ 梯度     │ │ 梯度     │ │ 梯度     │          │
│   │ 优化器   │ │ 优化器   │ │ 优化器   │ │ 优化器   │          │
│   │ GPU 0   │ │ GPU 1   │ │ GPU 2   │ │ GPU 3   │          │
│   └──────────┘ └──────────┘ └──────────┘ └──────────┘          │
│                                                                 │
│   ZeRO Stage 3：每张卡只存 1/N                                   │
│   ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐          │
│   │ 参数 1/4 │ │ 参数 2/4 │ │ 参数 3/4 │ │ 参数 4/4 │          │
│   │ 梯度 1/4 │ │ 梯度 2/4 │ │ 梯度 3/4 │ │ 梯度 4/4 │          │
│   │ 优化器1/4│ │ 优化器2/4│ │ 优化器3/4│ │ 优化器4/4│          │
│   └──────────┘ └──────────┘ └──────────┘ └──────────┘          │
│                                                                 │
│   内存节省：N 倍！（N = GPU 数量）                                │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### ZeRO 三个阶段

| Stage | 分片内容 | 内存节省 |
|-------|----------|----------|
| ZeRO-1 | 优化器状态 | 4x |
| ZeRO-2 | 优化器状态 + 梯度 | 8x |
| ZeRO-3 | 优化器状态 + 梯度 + 参数 | N x |

## 混合精度训练

**核心思想：** 用更小的数字格式，节省内存和加速计算。

| 精度 | 位宽 | 范围 | 用途 |
|------|------|------|------|
| FP32 | 32 bit | 最大 | 传统训练 |
| FP16 | 16 bit | 较小 | 主计算 |
| BF16 | 16 bit | 和 FP32 一样 | 推荐！ |
| INT8 | 8 bit | 最小 | 推理 |

**为什么 BF16 更好？**
- FP16 容易溢出（数字太大存不下）
- BF16 保持和 FP32 一样的范围，只是精度低一点

## 实际训练配置示例

| 模型大小 | GPU 配置 | 并行策略 |
|----------|----------|----------|
| 7B | 8 × A100 | 数据并行 + ZeRO-2 |
| 70B | 64 × A100 | 3D 并行（DP + TP + PP） |
| 175B+ | 数百张 | 3D 并行 + ZeRO-3 |

## 总结

| 概念 | 通俗理解 |
|------|----------|
| 数据并行 | 多人做同样的事，处理不同数据 |
| 张量并行 | 把一个大任务拆给多人同时做 |
| 流水线并行 | 流水线作业，每人负责一段 |
| ZeRO | 分片存储，不再重复保存 |
| 混合精度 | 用小一点的数字，省内存 |
